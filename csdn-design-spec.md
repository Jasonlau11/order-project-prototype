## 1. 设计原则与品牌调性

- **设计原则**：一致性、清晰、效率、可访问性等，作为所有界面决策的顶层依据。
- **品牌调性**：产品整体气质（如专业、简洁、可信、易用等），在视觉与交互上的体现方式。
- **整体风格**：**简约高级**。大量留白，视觉元素简洁，色彩搭配克制且重点突出；**内容优先**，设计服务于内容，确保信息易于扫描和阅读。
- **模块划分**：模块与模块之间以**间距**为主、**分割线**为辅，避免过多线框与装饰，保持界面干净、留白合理。
- **一致性**：所有组件、字体、颜色、间距均需遵循统一规范，确保跨页面、跨模块体验一致。
- **与现有产品关系**：与 CSDN 主站/创作中心等既有产品的设计语言保持一致，便于用户心智统一。
- **组件库约束**：页面中的元素应**优先使用现有 VEUI 组件库中的对应组件**进行设计；若 VEUI 已提供按钮、输入框、表格、分页、弹窗、标签等组件，设计稿中应直接采用或基于其样式做主题定制（如主题色 #fc5531、胶囊按钮等），以便设计与开发实现一致、减少重复造轮。

## 2. 布局与栅格

- **整体布局**：采用**三栏布局**——**左侧固定导航栏**、**中间主内容区**、**右侧辅助信息/推荐区**（若存在）。页面内容以**卡片或列表**形式模块化展示，模块之间通过充足的垂直间距做视觉分隔。
- **页面结构**
  - **顶栏（固定头部）**：含品牌 Logo、产品标题（如「创作中心」）、主操作按钮（如「发布」「创作」）、用户相关入口（头像、消息等）。背景与页面一致为纯白。
  - **左侧导航栏**：固定于左侧，支持一级与二级导航；可展开/收起子项，子项相对父级有缩进。
  - **主内容区**：占据主要视窗，用于列表、表格、图表、表单等核心内容。
  - **右侧区（可选）**：推荐列表、快捷入口等，与主内容区通过间距或细分割线区分。
- **栅格系统**：列数、水槽（Gutter）、边距（Margin）、断点（Breakpoint）与栅格使用规则。
- **安全区域**：Web 与移动端的安全区域、折叠屏/刘海屏等考虑。

## 3. 字体与排版

- **字体族**：优先使用系统默认无衬线字体，以保证多平台兼容与一致。推荐：**PingFang SC**、**Helvetica Neue**、**Arial** 等，中英文 Fallback 需在实现时统一。
- **字号阶梯**
  - **一级标题/页面标题**：18px – 20px。
  - **内容标题/卡片标题**：16px – 18px。
  - **正文/描述文本**：13px – 14px。
  - **辅助信息/注释/表格次要信息**：12px。
- **字重（Font Weight）**
  - **标题、重要信息、导航选中项**：中等或粗字重（`500` 或 `600`）。
  - **正文、普通文本**：常规字重（`400`）。
- **行高与字间距**：文本行高适中，建议为字体大小的 **1.5 – 1.8 倍**，以保证阅读舒适；字间距按默认或略紧即可。
- **排版层级**：通过字号、字重、颜色（见第 4 章）共同体现信息层级，保持标题醒目、正文可读、辅助信息弱化。

## 4. 色彩规范

- **页面背景**：统一为 **#ffffff**（顶栏、侧栏、主内容区一致）。
- **主题色（橙红色）**
  - **色值**：**#fc5531**（甲方确认的品牌主色）。
  - **使用原则**：在页面中**不可使用过多**，仅用于关键操作、选中强调等少量场景，保持简约高对比。
  - **典型用法**：主操作按钮背景（如「+ 创作」）；导航栏选中项的**左侧竖条指示**（约 3px 宽）；主内容区页签选中时的**下划线**；部分筛选/视图切换选中时的背景与文字（如「今日」「趋势图」）；图表主数据线；用户中心/会员中心等入口强调；**列表内操作链接**（如「去创作>」「去参与>」「去上传>」）及小图标强调；Logo。
- **中性色**
  - **文字**：一级标题/重要文本 **#222222** 或 **#333333**；正文/次要信息 **#666666**；辅助/描述性文字 **#999999**；禁用状态 **#cccccc**。
  - **边框/分割线**：极浅灰 **#eeeeee** 或 **#f0f0f0**，表格行分隔等。
- **辅助色与交互**
  - **链接/可点击文本**：默认深灰或中灰；**Hover 状态**为蓝色，如 **#409EFF** 或相近蓝色。
  - **导航栏选中背景**：浅灰 **#f0f0f0**、**#f5f5f5**、**#f7f8fa** 或 **#F6F7F9**；左侧主题色竖条 **#fc5531**。
  - **筛选栏/页签**：**未选中**——灰色背景（如 **#e0e0e0**）、黑色文字；**选中**——黑色背景（**#000000** 或 **#333333**）、白色加粗文字。
- **语义色（状态）**
  - **成功/通过**：绿色文字，标签可用浅绿背景 + 绿色文字（如「审核通过」）。
  - **警告/未通过**：主题色 **#fc5531** 或浅橙红背景 + #fc5531 文字（如「审核未通过」）。
  - **错误/信息**：按通用语义色约定，与主色区分。
- **深色模式**：是否支持、色板与切换规则（若适用）。

*（语义色精确色值可随甲方品牌色板继续补充）*

## 5. 组件与模式

- **组件库优先**：设计页面中的按钮、输入框、选择器、表格、分页、弹窗、标签、导航等元素时，**应使用现有 VEUI 组件库中的对应组件**；AI 输出或 Figma 组件命名宜与 VEUI 组件名或用法对应，便于开发直接复用。本规范中的样式描述（如胶囊按钮、主题色、选中状态）在落地时以 VEUI 组件的主题配置或样式覆盖方式实现，保持视觉一致。
- **基础组件**
  - **按钮（Button）**
    - **形状**：**胶囊形**（四角统一大圆角，药丸状）；宽大于高，内边距充足，内容垂直居中。
    - **主按钮**
      - **默认状态**：背景 **#fc5531**，无边框、无阴影，扁平风格。左侧白色加号图标（+）+ 白色文案（如「创作」）+ 右侧白色**向下箭头**（chevron），表示可展开下拉；图标与文案之间留白清晰。
      - **Hover 状态**：颜色较默认**轻微变深或变灰**，以区分可点击，不与默认完全一致；可带下拉展开；若为**带下拉的主按钮**，点击后箭头变为**向上箭头**表示菜单已展开。
      - **按下 / 点击**：**无单独按下样式**；保持 hover 样式，随后**直接跳转新页面**或**展开下拉菜单**，不做按下态视觉变化。
      - **下拉菜单（可选）**：主按钮支持下拉时，菜单为**白底**、**圆角与按钮上缘对齐**、**轻微阴影**与页面区分；列表项为**黑色图标 + 黑色文字**，每项统一内边距，图标在左、文案在右。
    - **次要/幽灵按钮**：透明背景，边框与文字为中性灰（如「关注」「导出数据」）；可为浅灰胶囊背景 + 深灰文字。**Hover** 时颜色较默认**轻微变灰或变深**，不与默认一致。
  - **输入框（Input）**：边框小圆角（4px – 6px）；背景浅灰或白色，边框浅灰；日期选择等与整体风格一致。
  - 选择器、开关、标签、分页、弹窗、提示等需保持默认、悬停、聚焦、禁用、加载、错误等状态一致。
- **业务组件**：订单卡片、状态标签、里程碑时间线、账单列表等与订单业务强相关的组件，在视觉与交互上需与上述基础组件及导航/列表/表格/状态标签等规范一致。
- **导航与信息架构**
  - **顶栏**：见第 2 章布局；主操作按钮为橙红胶囊，其余图标与文字为深灰/黑色。
  - **左侧导航栏**
    - **未选中**：透明背景，黑色/深灰字体，图标为**线条（Outline）**风格。
    - **选中（一级）**：浅灰背景（如 #F6F7F9），黑色**加粗**字体，图标可变为**填充（Filled）**；**左侧 3px 左右宽的主题色竖条**作为选中强调。
    - **选中（二级子项）**：浅灰背景，黑色加粗字体；无竖条时可仅靠背景与字重区分。
  - **主内容区页签**（如「博客数据」「下载数据」）
    - **未选中**：透明背景，深灰文字。
    - **选中**：深灰/黑色加粗文字，**主题色下划线**。
  - **筛选栏/标签（Filter / Tag）**
    - **未选中**：灰色背景（如 **#e0e0e0**），黑色字体。
    - **选中**：黑色背景（**#000000** 或 **#333333**），白色加粗字体。
  - **分段控件/视图切换**（如「近7天」「近1月」「趋势图」「数据列表」）
    - **未选中**：透明背景，深灰文字与图标。
    - **选中**：可为浅灰背景 + 深灰文字，或**浅橙红背景 + 主题色文字与图标**（用于强调当前视图）。
- **列表与卡片**
  - **内容结构**：标题、摘要/描述、元数据（阅读量、评论数等）分层清晰。
  - **视觉分隔**：列表项之间以**垂直间距**为主，可辅以极细浅灰分割线（1px–2px）；卡片与模块间留白充足。
  - **可跳转元素 Hover**：标题、链接等可点击元素在 **hover** 时字体变为**蓝色**（如 #409EFF）。
  - **操作链接**：列表/卡片内的行动号召链接（如「去创作>」「去参与>」）使用**主题色 #fc5531**，与正文链接的 Hover 蓝区分。
- **卡片/面板（Card / Panel）**：内容块为白底（#ffffff）、**圆角 4px–8px**，通过**轻微阴影**或**极细浅灰边框**与背景区分，避免粗边框；模块间以留白为主。
- **数据卡片（如汇总数据）**：浅灰背景、小圆角（4px–6px）；指标名称小号浅灰，数值大号加粗深色；可带「i」等说明图标。
- **徽标/标签（Badge / Label）**：小尺寸信息标签（如「LV.1」「博客」「专栏」）使用**浅色背景 + 深色文字**，圆角约 **2px–4px**，与按钮、状态标签区分。
- **表格（Table）**
  - **表头**：黑色/深灰文字，常规字重，垂直居中。
  - **行**：每行之间用极细浅灰横线分隔（#F0F0F0 等）；**不使用**交替行背景色，**不使用**列间竖线，依靠留白与对齐区分列。
  - **内容**：左对齐；次要信息（如日期、原因说明）可用小号浅灰，括号内说明紧跟状态标签。
- **状态标签（Status Tag / Pills）**
  - **形状**：小圆角矩形「药丸」状，圆角约 4px – 8px。
  - **审核未通过**：浅橙红背景 + 橙红文字。
  - **审核通过**：浅绿背景 + 绿色文字。
  - 原因等说明文字：小号浅灰，括号内紧跟标签后。
- **图表（Chart）**：主数据线使用主题色；图表区域为白底、极细浅灰边框、轻微阴影，与背景区分；图例可用小色块与勾选控制显隐。
- **滚动条**：细而浅灰，与主内容区右侧对齐，风格简约。
- **弹窗（Modal / Dialog）**
  - **标准状态**：白底（#ffffff）、**圆角约 8px–10px**、**轻微阴影**，居中于**浅灰遮罩**之上；整体扁平、简约。
  - **头部**：标题加粗、深灰/黑色，左对齐；右上角 **×** 关闭图标，深灰/黑色，用于关闭弹窗。
  - **内容区**：确认类文案（如「确定要删除该文章吗?」）常规字重、深灰，左对齐；与标题之间留白清晰。
  - **底部操作**：两个按钮**右对齐**排列。**次要按钮**（如「取消」）在左、**主要按钮**（如「确定」）在右。
  - **次要按钮**（如「取消」）：**默认**——白底、细浅灰边框（约 1px）、深灰文字、小圆角。**Hover**——颜色较默认**轻微变灰或变深**（如背景略灰、边框略深），不与默认一致。
  - **主要按钮**（如「确定」）：**默认**——背景 **#fc5531**、白色文字、无边框、小圆角。**Hover**——颜色较默认**轻微变深或变灰**，以区分可点击，不与默认一致。
  - **通用**：弹窗内按钮在 Hover 时均需有视觉反馈（轻微变灰或变深），不可与默认状态完全相同。
- **空状态（列表无数据）**
  - **布局**：主内容区在无数据时保留顶部筛选/搜索栏（如「全部分类」「已发布」「请输入关键词」「筛选」等）；下方大面积空白区域用于展示空状态内容。
  - **文案与按钮**：在空白区域**水平与垂直居中**展示**浅灰文案**「暂无内容」；正下方为**主操作按钮**——主题色 **#fc5531** 背景、白色文字、圆角、左侧带白色图标（如笔形），文案与业务一致（如「写文章」），作为创建内容的入口。
  - **风格**：简洁、不喧宾夺主；无数据时仍保持页面框架（顶栏、侧栏、筛选栏），仅列表区域为空状态。
- **加载状态（数据加载中）**
  - **表现**：数据请求期间**页面内容区为空**（或保持上一帧），不先展示骨架屏；在**浏览器窗口顶部**显示一条**主题色（#fc5531）窄进度条**，由**左至右**填充。
  - **结束**：进度条**填充完毕后消失**，随后下方页面内容展示出来；无额外全屏遮罩或居中 Loading 图标（除非业务另有要求）。
- **错误页**：暂无甲方参考页面；待提供 404、网络错误等错误页截图后再补充规范。
- **反馈与提示**：Toast、Message、Drawer 等与整体简约风格一致，可参考弹窗的遮罩与圆角。

## 6. 间距与密度

- **间距体系**：基础间距单位建议 4px/8px；组件内边距、模块间距、页面边距按阶梯使用。
- **模块间距**：不同功能模块或内容卡片之间保持**较大垂直间距**，如 **24px – 32px**；少量情况可用细分割线辅助分隔。
- **内容内间距**：卡片内部、文本与边框之间统一内边距，如 **16px – 20px**。
- **模块划分**：模块与模块之间以**间距**为主、**少量分割线**为辅，整体风格简约高级；卡片/面板可配合**轻微阴影**与背景区分，避免粗边框。
- **密度模式**：紧凑/标准/宽松的适用场景与具体数值可按业务需要再定。

## 7. 图标与图形

- **图标风格**：采用简洁的**线条（Outline）**风格；**圆角统一为 1px**；线宽、尺寸需统一规范。

## 8. 动效与过渡

- **动效原则**：用于反馈、引导、状态过渡；保持克制，不喧宾夺主。
- **Hover 效果**：可点击元素（链接、按钮、导航项）需有清晰 Hover 反馈，如文字变色（蓝）、背景变化或轻微阴影。
- **点击反馈**：点击后可有按压状态、加载指示等视觉反馈；选中状态需明确区分当前活动元素。
- **时长与缓动**：常用动效时长（如 200ms/300ms）、缓动曲线（ease-in-out 等）。
